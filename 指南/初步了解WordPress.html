初步了解WordPress ​

WordPress 是一个非常流行的开源内容管理系统（CMS），用于创建和管理网站。它最初被设计为一个博客平台，但随着时间的发展，已经成为一个功能强大且极其灵活的CMS，可以用来构建各种类型的网站，包括个人博客、企业网站、电子商务平台、新闻门户、论坛等等。

设置 ​

几个注意事项。

  • 常规 包括网站标题、副标题、时区、语言等基本信息的设置。如果做成外贸站，建议选择English(United States)。如果有备案，建议网站标题和备案的信息一致。
  • 固定链接 看主题是否有硬性要求，通常选择文章名，及后缀为%postname%，要在自定义结构中添加。样式举例，一个标题为Privacy Policy页面，网址通常为example.com/Privacy-Policy，如果这个页面的父级是About US，则example.com/abdout-us/privacy-policy。
  • Akismet反垃圾评论 请在第一时间注册Akismet的免费账号，并输入API密钥。
  • 阅读 设置对搜索引擎的可见性，一般建议不勾选；也可以等到网站建设好了之后，在设置SEO时不勾选。
  • Redis 如果有阅读前面关于Redis应用，设置中可能多了一个Redis选项。它可以在 WordPress 网站中显著提高性能，加快页面加载速度和提高整体用户体验。

用户 ​

WordPress 支持多种用户角色，每种角色具有不同的权限。用户角色的设定旨在确保网站的安全性和功能性，让不同的用户能够在限定的范围内操作网站而不影响其他重要功能或数据。以下是 WordPress 中的用户角色及其对应的权限：

  • Administrator（管理员） 拥有对整个网站的完全控制权，可以执行任何操作，包括安装插件和主题、管理用户、编辑网站设置等。
  • Editor（编辑） 可以编辑和发布所有的文章和页面，包括其他用户的文章。还可以管理评论。
  • Author（作者） 可以发布和编辑自己的文章，但不能编辑其他用户的文章。
  • Contributor（贡献者） 可以撰写和编辑自己的文章，但不能发布文章。需要编辑或管理员审核后才能发布。
  • Subscriber（订阅者） 只能登录网站并查看个人资料。可以发表评论。

文章 ​

在 WordPress 中，文章（Post）是一种常用的内容类型，主要用于发布和管理动态内容，如博客文章、新闻更新、日记条目等。文章是 WordPress 最基本也是最核心的内容形式之一，具有多种用途和功能。

媒体 ​

在 WordPress 中，媒体（Media）是指上传到网站上的各种文件，主要包括图片、音频、视频以及其他文件类型。媒体库（Media Library）是 WordPress 提供的一个功能强大的工具，用于管理和组织这些文件。媒体库的存在极大地简化了网站内容的管理和展示。

图片使用建议 ​

图片类请使用WebP格式，WebP 格式支持有损和无损压缩，同时还支持透明度（Alpha 通道），WebP 相比传统的 JPEG 和 PNG 格式，能够在保持相同视觉质量的情况下，提供更小的文件体积。

如何将图片格式转换为WebP，批量的可以使用 格式工厂。

图视频使用建议 ​

视频建议使用社交媒体，然后做外链处理。国外常见媒体是YouTube，国内常见媒体是腾讯视频、优酷、西瓜视频等等，不推荐B站，B站会限制清晰度。

其他使用建议 ​

关于LOGO，建议使用SVG格式，或者使用 iconfont代码应用

页面 ​

在 WordPress 中，“页面（Page）”是一种用于创建和管理静态内容的内容类型。与“文章（Post）”不同，页面通常用于展示固定不变的信息，如“关于我们”、“联系我们”、“服务介绍”等。页面的设计和功能使得它们非常适合用于构建网站的核心部分。

如果有安装主题，一般可以使用主题自带的模板或页面构建器创建页面

外观 ​

主题 ​

默认有4个主题，分别是2021，2022，2023和2024，每年新增一个。主题决定了网站的视觉风格、页面结构和一些基本功能，是网站前端展示的核心。一个主题通常包含一组 HTML、CSS 和 JavaScript 文件，以及图像和其他资源文件。

主题推荐 ​

主题通常会自带插件或模板，有不同于默认主题的视觉风格、页面结构和一些基本功能。如果单纯使用页面构建起插件，而不是用主题，字体，字体间距等参数仍然受默认主题的影响。

子主题和父主题 ​

在 WordPress 中，父主题（Parent Theme）和子主题（Child Theme）是一种主题架构方式，用于在不修改原有主题代码的情况下进行自定义和扩展。这种架构方式使得用户可以在保留原有主题功能的同时，进行个性化定制，同时保证升级原有主题时不会丢失自定义的内容。 优设 就是基于B2进行个性化定制，建设为自己独有的子主题。

额外css ​

在 WordPress 中，"额外 CSS"（Additional CSS）通常指的是在现有主题样式的基础上添加自定义的 CSS 代码，以实现特定的样式调整或功能增强。这种方法使得用户可以在不修改主题文件的情况下进行个性化的样式定制。

举例：

css
/*小图标*/
.icon {
   width: 1.2em; height: 1.2em;
   vertical-align: -0.15em;
   fill: currentColor;
   overflow: hidden;
}
/*小图标结束*/

上述代码的作用是自定义了icon大小。

小工具 ​

在 WordPress 中，“外观-小工具”（Appearance > Widgets）是一个重要的功能，它允许用户在网站的不同区域（如侧边栏、页脚、头部等）添加和管理各种功能性的组件，这些组件被称为“小工具”（Widgets）。小工具可以提供多种功能，从简单的文本块到复杂的插件功能展示。

前端主要位置：侧边栏（Sidebar）

菜单 ​

“外观-菜单”（Appearance > Menus）是一个功能强大的工具，用于创建、管理和自定义网站的导航菜单。导航菜单是网站的重要组成部分，它帮助用户快速找到所需的内容，并提高网站的可用性和用户体验。

前端主要位置：导航栏

插件 ​

WordPress 插件是一种扩展 WordPress 功能的软件组件，它允许用户在不修改核心代码的情况下为网站添加新的功能或增强现有功能。插件是 WordPress 生态系统的重要组成部分，极大地丰富了 WordPress 网站的功能性和灵活性。

后记 ​
